02Why widget-first

조립 대신 완제품.
그게 머천트의 시간을 돌려줍니다.

전통적인 결제 SDK는 컴포넌트를 머천트가 직접 조립합니다. OpenCheckout은 네 개의 검증된 위젯을 그대로 마운트하면 되는 위젯-우선 표면입니다. 상태 동기화, 검증, 재시도, 보안 경계는 위젯이 책임집니다.

04Integrate in five minutes

설치 없이, 네 단계로 끝납니다.

CDN 한 줄, 마운트 네 개, 주문 컨텍스트 설정, requestPayment. 샌드박스 키는 페이지가 자동으로 주입합니다.

SDK 로드

CDN에서 ES 모듈로 가져옵니다. 번들러도 npm도 선택 가능합니다.

import { OpenCheckout } from "https://cdn.opencheckout .dev/v1";

세션 생성

publishableKey와 customerKey로 widgets 인스턴스를 만듭니다.

const oc = await OpenCheckout .load({ publishableKey }); const widgets = oc.widgets({ customerKey, locale });

네 개 마운트

setAmount + setOrder 후, 네 위젯을 각자 selector에 마운트.

widgets.renderAddress({…}); widgets.renderShipping({…}); widgets.renderPayment({…}); widgets.renderAgreement({…});

결제 요청

버튼 한 번으로 successUrl·failUrl로 리다이렉트합니다.

widgets.requestPayment({ successUrl, failUrl, });

05Security by default

보안은 옵션이 아니라 기본값입니다.

카드번호(PAN)는 머천트 코드에 닿지 않습니다. 위젯은 Shadow DOM 안에서 격리되며, customerKey는 항상 UUID로 발급되고, requestPayment 페이로드는 서버 사이드 검증을 거칩니다.

PAN guard

PAN 패턴이 metadata에 섞이면 마운트 단계에서 차단합니다. 머천트 콘솔에는 절대 흐르지 않습니다.

Shadow DOM 격리

위젯 내부 트리는 머천트 페이지의 CSS·JS에서 분리됩니다. 의도치 않은 스타일 누수도 차단됩니다.

customerKey UUID

익명 세션에도 UUID v4를 발급해 사용자 추적 가능성을 최소화합니다. ANONYMOUS는 데모 전용 별칭입니다.

이벤트 우선 통합

위젯 → 머천트 코드 흐름은 단방향 이벤트. polling, 양방향 RPC, 글로벌 상태가 없습니다.

Idempotent requestPayment

같은 orderId에 대한 중복 호출은 서버에서 멱등 처리. 클라이언트 재시도가 안전합니다.

Webhook 서명 검증

모든 결제 이벤트는 서명된 webhook으로 머천트 서버에 도달합니다. 재생 공격은 timestamp tolerance로 차단.
